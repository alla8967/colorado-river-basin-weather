# Project Map

This repository contains both source code and local generated artifacts. For a
human review, treat it as a source project with a few ignored artifact shelves
that may be physically present in the working folder.

## Start Here

1. Read `README.md` for the project purpose, pipeline, and common commands.
2. Use `docs/reviewer_runbook.md` for the practical review path.
3. Run `make check` for the local app and C++ engine validation set.
4. Use `docs/research_script_inventory.md` when reviewing model/research scripts.
5. Use `docs/artifact_quarantine_plan.md` before moving, deleting, or staging
   generated outputs.

Each major source folder also has a local `README.md` that explains the purpose
of that collection before you open individual files.

## What To Ignore While Browsing

These folders and files can make the working directory look cluttered, but they
are local/generated byproducts and should not be staged:

- `.venv/`, `station-proxy-backend/.venv/`, `*.egg-info/`
- `.pytest_cache/`, `build/`, `__pycache__/`
- compiled C++ binaries such as `station_engine_api`,
  `validate_prediction_similarity`, and `Station_Engine_Server/station_engine_server`
- `NOAA_Inventory_Sort/NOAA_GHCN_ByYear/`
- `NOAA_Inventory_Sort/*_daily_app_ready.csv`
- `NOAA_Inventory_Sort/*_daily_recent_long_format.csv`
- `weather_reconstruction_model/cache/`
- `weather_reconstruction_model/outputs/`
- `weather_reconstruction_model/model_runs/`
- `weather_reconstruction_artifacts/`
- `terrain_data/raw_dem/`, `terrain_data/processed/`, and local `Raw_DEM/*.tif`

Some of these ignored files are still useful for local runs. Do not move or
delete data/model artifacts just to make Finder look cleaner unless you have a
clear regeneration path.

## Mental Model

The project has four main lanes:

- App lane: `station-proxy-backend/`, `C++_Weather_Station_Proxy_Engine/`,
  `Station_Engine_Server/`, and `tests/`.
- Data-prep lane: `NOAA_Inventory_Sort/`.
- Research/model lane: `weather_reconstruction_model/scripts/`.
- Remote-run lane: `remote_jobs/`.

The repository root is intentionally kept small: project overview, map, build
metadata, and environment examples stay at the top level; detailed runbooks live
under `docs/`, and folder-specific explanations live beside the code they
describe.

The cleanup work is intentionally improving boundaries inside those lanes
without doing a risky top-level reshuffle.

## Safe Local Commands

```bash
make check
PYTHON=.venv/bin/python make test-python
make clean-local-artifacts
git status --ignored --short
```

`make clean-local-artifacts` is intentionally conservative. It removes
rebuildable compiled/test byproducts only; it does not delete NOAA archives,
model runs, DEM data, or research outputs.

## Alpine Safety Boundary

Active or pending station-holdout jobs may be using the copied code under:

```text
/scratch/alpine/$USER/crb_weather_runs/current/weather_reconstruction_model/scripts
/scratch/alpine/$USER/crb_weather_runs/current/remote_jobs
```

Do not rsync, sync, rename, or reorganize that Alpine scratch copy while those
jobs are running. This local cleanup branch can change local files, but Alpine
scratch remains frozen run code until a new version is validated and
intentionally synced.
