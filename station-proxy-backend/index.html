Methodology

The app finds the nearest available station first. If that station is target-only, it ranks proxy hub stations; if that station is already a hub, it ranks the most similar other hubs.

Target stations

Target stations are recent stations that can represent the clicked point when the nearest available station is not already a hub.

  • Must have both TMAX and TMIN records.
  • Must be inside the study bounds: 31.0-43.5 latitude and -115.0 to -102.0 longitude.
  • Must have at least 20 usable overlapping years of TMAX and TMIN.
  • Must have usable temperature coverage ending in 2020 or later.
  • Target-only stations are compared against hub stations when they are the nearest station to the clicked point.
Hub stations

Hub stations are longer-record stations that can act as proxy options, or as the nearest station itself when a clicked point lands closest to a hub.

  • Must meet the same TMAX, TMIN, recency, and geographic bounds rules as target stations.
  • Must have at least 42 usable overlapping years of TMAX and TMIN.
  • Must have usable temperature coverage dating to 1960 or earlier.
  • The current hub set is generated directly from the NOAA inventory using these record-length rules.
  • If the nearest station is a hub, the app compares it with the five most similar other hubs.
Proxy ranking filters

Before ranking, proxy candidates are screened so distant or weakly supported matches do not dominate the map.

  • Maximum target-to-proxy distance: 300 km.
  • Maximum elevation difference: 1500 m.
  • Minimum paired daily observations: 365 days for full datasets.
  • Stations with missing daily correlation, MAD, or RMSE are excluded.
Scoring metrics

The final score is a weighted 0-100 proxy score. Higher is better.

  • Daily correlation measures whether the two stations move together through time.
  • Daily MAD measures average absolute temperature difference.
  • Daily RMSE penalizes occasional large mismatches more strongly.
  • Distance and elevation help keep physically plausible stations near the top.
  • Monthly metrics are reported for context, but the current score is driven by daily metrics plus geography.

Daily correlation

Correlation is calculated as Pearson correlation on paired daily average temperatures.

\[ r = \frac{\sum_i (T_i - \bar{T})(P_i - \bar{P})} {\sqrt{\sum_i (T_i - \bar{T})^2} \sqrt{\sum_i (P_i - \bar{P})^2}} \] \[ S_r = \begin{cases} 1, & r \ge 0.95 \\ 0, & r \le 0.75 \\ \frac{1 + \cos(\pi \cdot \frac{0.95-r}{0.95-0.75})}{2}, & 0.75 < r < 0.95 \end{cases} \]
r = cov(target, proxy) / (sd(target) * sd(proxy)) score = 1.0 if r >= 0.95
score = 0.0 if r <= 0.75
otherwise cosine-smoothed between 0.75 and 0.95

The cosine curve avoids a harsh cliff: a station does not suddenly become bad at 0.949, but very weak relationships still fall away quickly.

Daily MAD

Mean absolute difference captures the typical day-to-day temperature offset.

\[ MAD = \frac{1}{n}\sum_i |T_i - P_i| \] \[ S_{MAD} = \begin{cases} 1, & MAD \le 2.0 \\ e^{-0.12(MAD - 2.0)}, & MAD > 2.0 \end{cases} \]
MAD = mean(abs(target_tavg - proxy_tavg)) score = 1.0 if MAD <= 2.0 F
score = exp(-0.12 * (MAD - 2.0)) otherwise

This gets the largest weight because a proxy should be close in actual temperature, not just move in the same direction.

Daily RMSE

RMSE is similar to MAD, but larger mismatches are penalized more strongly.

\[ RMSE = \sqrt{\frac{1}{n}\sum_i (T_i - P_i)^2} \] \[ S_{RMSE} = \begin{cases} 1, & RMSE \le 2.75 \\ e^{-0.13(RMSE - 2.75)}, & RMSE > 2.75 \end{cases} \]
RMSE = sqrt(mean((target_tavg - proxy_tavg)^2)) score = 1.0 if RMSE <= 2.75 F
score = exp(-0.13 * (RMSE - 2.75)) otherwise

This helps reject stations that usually look fine but occasionally diverge in a way that would be risky for proxy use.

Distance

Distance is calculated with the haversine formula between the target and proxy coordinates.

\[ a_1 = \sin^2\left(\frac{\Delta\phi}{2}\right) \] \[ a_2 = \cos(\phi_1)\cos(\phi_2) \sin^2\left(\frac{\Delta\lambda}{2}\right) \] \[ a = a_1 + a_2 \] \[ d = 2R\arctan2(\sqrt{a}, \sqrt{1-a}) \] \[ S_d = \begin{cases} 1, & d \le 35 \\ e^{-0.006(d - 35)}, & d > 35 \end{cases} \]
distance = great-circle km between stations score = 1.0 if distance <= 35 km
score = exp(-0.006 * (distance - 35)) otherwise

Distance is not allowed to dominate temperature similarity, but nearby stations are more likely to share storm tracks, terrain exposure, and regional patterns.

Elevation

Elevation difference is the absolute vertical separation between the two stations.

\[ \Delta z = |z_{target} - z_{proxy}| \] \[ S_z = \begin{cases} 1, & \Delta z \le 50 \\ e^{-0.0025(\Delta z - 50)}, & \Delta z > 50 \end{cases} \]
elevation_diff = abs(target_elevation - proxy_elevation) score = 1.0 if elevation_diff <= 50 m
score = exp(-0.0025 * (elevation_diff - 50)) otherwise

Elevation matters in mountain regions, but it gets the smallest weight because good climate matches can still exist across imperfect elevation pairs.

Final score

Each normalized metric becomes a 0-1 subscore. The weighted result is multiplied by 100.

\[ Score = 100( 0.35S_{MAD} + 0.25S_{RMSE} + 0.20S_r + 0.15S_d + 0.05S_z) \]
final = 100 * (0.35*MAD + 0.25*RMSE + 0.20*r + 0.15*distance + 0.05*elevation)

Record overlap is used as a quality gate rather than a direct score component, because targets and hubs are intentionally allowed to have different record lengths.
